# データ取得手順（DStockAnalysis）

本書は、銘柄データ（全銘柄リスト＋各指標の実数値）をアプリに取り込む手順をまとめたものです。
**ローカル単独動作・無料・登録不要**を前提とし、できる限りバッチで簡略化しています。

---

## 0. データの全体像（重要）

| 区分 | 内容 | 取得方法 | 自動化 |
|---|---|---|---|
| **全銘柄リスト** | コード・銘柄名・市場・業種・規模 | JPX「東証上場銘柄一覧」(公式・無料・登録不要) | ✅ バッチ/アプリで自動 |
| **主要指標(数値)** | PER・PBR・ROE・配当利回り・時価総額・EPS 等 | IR BANK / みんかぶ / 株探 等から取得 | ✅ `fetch_data.bat`(週1・低速) |
| **その他(財務/CF/優待 等)** | 自己資本比率・営業CF・優待内容 等 | サイトのリンクで確認 → CSV | △ 半自動(テンプレ＋転記) |

> **取得方針(規約・サーバ負荷への配慮)**: 本ツールは対象サイトの **robots.txt を必ず確認**し、
> 許可されたページのみを **十分な間隔(既定8秒+ゆらぎ)** を空けて取得します。短期売買用ではないため
> **週1回程度の低頻度**で運用し、対象は **ウォッチリスト(`codes.txt`)の銘柄に限定**します。
> robots.txt が自動アクセスを拒否している **バフェット・コードは取得対象外**(リンクのみ)です。
> 取得値は各サイトの表示基準・更新タイミングにより差があり **ベストエフォート**です。CSV を取り込む前に
> 内容を確認・修正できます。
> （完全自動で全指標・過去履歴まで取得したい場合は登録制の J-Quants/EDINET が必要ですが、
> 過去情報は有料のため本構成では使用しません。）

---

## 1. 全銘柄リストの取得（自動）

### 方法A: バッチで更新（アプリ起動前でもOK・推奨）
1. `tools\update_master.bat` を**ダブルクリック**。
2. JPX 公式の `data_j.xls` が `%AppData%\DStockAnalysis\data_j.xls` に保存されます。
3. アプリを起動すると、最新の全上場銘柄（約3,700）が自動で読み込まれます。

### 方法B: アプリのボタンで更新
- アプリ右上の **「全銘柄更新(JPX)」** を押す（ネット接続が必要）。

> JPX 一覧は**毎月更新**（月末営業日基準）。データ更新日が古いと右上に「(更新を推奨)」が出ます。
> 取得元: `https://www.jpx.co.jp/markets/statistics-equities/misc/tvdivq0000001vg2-att/data_j.xls`

この時点で、起動直後から全銘柄が一覧表示され、業種・市場・規模での絞り込みは**実データ**で機能します。
PER 等の指標は初期状態では**サンプル(擬似)値**です（画面右下に件数表示）。

## 2. 主要指標を自動取得する（fetch_data.bat）

対象6サイトのうち robots.txt で許可された **IR BANK / みんかぶ / 株探 / 株マップ** から、
PER・PBR・ROE・配当利回り・時価総額・EPS などを取得して CSV を作ります。**Python が必要**（pip 不要・標準ライブラリのみ）。

### 手順 2-1: Python を用意（初回のみ）
1. https://www.python.org/downloads/ から Python をインストール。
2. インストール時に **「Add python.exe to PATH」にチェック**。
3. 確認: `python --version` でバージョンが出ればOK。

### 手順 2-2: 対象銘柄を決める
- `tools\codes.txt` に1行1銘柄コードで記入（初回実行時にサンプルが自動生成されます）。
- 注目銘柄だけにするのが安全・高速です（多すぎるとサイト負荷大。200超で警告）。

### 手順 2-3: 取得を実行
- `tools\fetch_data.bat` を**ダブルクリック**。
  - robots.txt を確認し、間隔(既定8秒)を空けて取得します。8銘柄で約1〜2分程度。
  - 6日以内に取得済みの銘柄はスキップ（週次運用向けキャッシュ）。`fetch_data.bat --force` で全件再取得。
  - 動作確認: `fetch_data.bat --test 7203`（1銘柄だけ表示・保存なし）。
  - 株探も使う: `fetch_data.bat --kabutan`（Crawl-delay 3秒を順守）。
  - 間隔変更: `fetch_data.bat --delay 12`。
- 出力: `tools\output\stocks_real.csv`

### 手順 2-4: アプリに取り込む
1. アプリ右上 **「CSV取込」** で `tools\output\stocks_real.csv` を選択。
2. 銘柄コードで照合し、**取得できた列だけ**が実データで上書きされます（他は維持）。
3. **「保存」**で永続化。

### 手順 2-5: 週1回の自動実行（任意・Windows タスクスケジューラ）
1. 「タスク スケジューラ」→「基本タスクの作成」。
2. トリガー: 毎週（例：日曜 8:00）。
3. 操作: プログラムの開始 → `tools\fetch_data.bat` を指定。
4. これで週1回 `stocks_real.csv` が自動更新されます（取込はアプリ起動時に手動で実施）。

## 3. その他の指標を手で補う（半自動）

財務(自己資本比率/有利子負債)・キャッシュフロー・成長率・株主優待など、自動取得しない項目を補います。

### 手順 3-1: テンプレCSVを出力
1. アプリ右上 **「テンプレ出力」** を押す。
2. 保存先（例: ドキュメント）を選ぶと、`stocks_template.csv` が出力されます。
   - 全銘柄の **Code / Name / Market / Sector / Scale** が前埋め済み、指標列は空欄です。

### 手順 3-2: 実数値を調べて記入
- 調べたい銘柄の行に、実数値を記入します（空欄のままの列は取込時に上書きされません）。
- 数値の確認は、アプリの **個別銘柄分析画面のリンク**（IR BANK / バフェット・コード / 株探 / 株マップ /
  みんかぶ / 株主優待ガイド）から該当銘柄ページを開くのが簡単です。
- 複数銘柄をまとめて開きたい場合は、`tools\open_sources.bat 7203` のようにコードを渡して実行すると、
  その銘柄の各サイトが既定ブラウザで一括で開きます。

主な記入列（`sample_stocks.csv` も参照）:
`Price, PER, PBR, ROE, EPS, BPS, DividendYield, PayoutRatio, EquityRatio, InterestBearingDebtRatio,
OperatingMargin, NetProfitMargin, OperatingCF, FreeCashFlow, RevenueGrowth3Y, BenefitYield, TotalYield` など。

### 手順 3-3: 取り込む
1. アプリ右上 **「CSV取込」** を押し、記入した CSV を選択。
2. **銘柄コードで照合**し、**CSV に記入した列だけ**が実データで上書きされます
   （記入しなかった列は維持。新規コードは追加）。
3. スコアは自動で再計算されます。**「保存」**で `%AppData%\DStockAnalysis` に保存され、次回も再現します。

> 一度に全銘柄ぶんを埋める必要はありません。注目銘柄から少しずつ実データ化していく運用が現実的です。

## 4. 更新ルーティン（おすすめ）

- **毎週**: `tools\fetch_data.bat`（注目銘柄の主要指標を取得）→ アプリで「CSV取込」→「保存」。
  - タスクスケジューラに登録すれば取得は自動化できます（手順 2-5）。
- **毎月**: `tools\update_master.bat`（新規上場・市場変更などを反映）。
- 決算が出た銘柄は、必要に応じてリンクで確認し、その他指標をテンプレ/CSV で補完。

## 5. 付属ツール一覧

| ファイル | 役割 |
|---|---|
| `tools\fetch_data.bat` / `.py` | 6サイト(許可分)から主要指標を取得し `output\stocks_real.csv` を生成（robots順守・低速・週次） |
| `tools\codes.txt` | 取得対象のウォッチリスト（1行1コード） |
| `tools\update_master.bat` / `.ps1` | JPX 全銘柄一覧を最新化（実データ・自動・登録不要） |
| `tools\open_sources.bat` | 指定コードの外部データソースを一括で開く（転記の効率化） |
| アプリ「テンプレ出力」 | 全銘柄前埋めの記入用 CSV を出力 |
| アプリ「CSV取込」 | 記入/取得した列だけを実データで反映（列単位マージ） |
| `sample_stocks.csv` | CSV の列名サンプル |

### fetch_data が取得する主な列
`Price, PER, PBR, ROE, DividendYield, MarketCap, EPS`（サイトにより `PayoutRatio, EquityRatio` も）。
財務詳細・CF・成長率・株主優待は手順3で補完してください（株主優待は誤検出回避のため自動取得対象外）。

## 6. （発展）完全自動化したい場合

無料登録制の公式 API を使えば、指標の実数値まで自動取得できます（登録はご自身で）。

- **J-Quants API**: 上場情報・財務諸表・株価。`https://jpx-jquants.com/`
- **EDINET API**: 有価証券報告書等の法定開示（XBRL）。

実装方針:
`Services` に `IStockDataProvider`（例: `JQuantsService`）を追加し、`List<Stock>` を返すようにすれば、
`MainViewModel` から CSV 取込と同じ流れで全銘柄へ反映できます（本アプリは差し替え前提の構造）。
ご希望があればこの連携も実装します。
