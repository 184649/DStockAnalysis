## 3. Service

### 3.1 ScoreService.cs
- `Recalculate(Stock)`：6 軸→合成→OverallScore→Judgement の順で設定。
- スケール関数 `Scale(value, low, high, max)` / `ScaleInverse(...)` で線形補間（0..max にクランプ）。
- 各メソッド：
  - `CalcSafety`：自己資本比率・有利子負債比率・配当金残年数
  - `CalcGrowth`：各種増収率・利益/EPS 成長率
  - `CalcProfitability`：営業利益率・純利益率・ROE
  - `CalcReturn`：配当利回り・連続増配・配当性向品質・自社株買い・累進配当・優待（最大15、CF赤字/性向>90で×0.4）
  - `CalcEfficiency`：ROE・営業CFマージン
  - `CalcValuation`：PER/PBR/MIX の逆スケール
  - `CalcLongTerm`：安全性0.25+収益性0.20+還元0.15+FCF0.15+参入障壁0.125+10年需要0.125
  - `CalcRevaluation`：売上/営業/純利益/EPS 成長＋株価変化＋テーマ需要
  - `CalcBuffett`：配点（事業理解15/競争優位15/収益性15/CF15/財務15/還元10/割安10/暴落時5）。
    チェック結果を加減点（はい→加点、いいえ→減点、不明→中立）。優待は還元10内で小さく加点。
  - `CalcWantToBuy`：長期0.25+再評価0.15+バフェット0.25+割安0.10+興味0.10+事業理解0.05+分類0.10
- `DecideJudgement`：分類が除外/保留なら優先。バフェット≥80&長期≥75&買いたい≥75→最重要候補、ほか閾値で各カテゴリ。
- 補助：`PayoutQuality`（30–55% を満点、超高/超低で減点）、`ClassificationBonus`。

### 3.2 CsvImportService.cs
- `ImportFromFile(path)` / `Parse(content)`。
- `ParseCsv`：RFC4180 風（`"` エスケープ・改行/カンマ処理）で 2 次元配列化。
- ヘッダー→列インデックス辞書（OrdinalIgnoreCase）。行ごとに `S/D/I/B` ヘルパで取得。
- `ParseDouble`：`,`/`%`/`円`/`倍` 除去、`-`/`N/A`/全角ハイフン→0、Invariant 解析。
- `ParseBool`：`1/true/yes/y/○/◯/あり/採用/◎`→true。
- 補完：HasShareholderBenefit（列が無ければ ShareholderBenefit/BenefitContent から推定）、
  FreeCashFlow（FreeCF エイリアス）、TotalYield（配当+優待）、MixFactor（PER×PBR）。
- `Code` 空/空行はスキップ。文字コードは BOM 自動判定（既定 UTF-8）。

### 3.3 DataStorageService.cs
- コンストラクタ `DataStorageService(string? baseDir=null)`：既定 `%AppData%\DStockAnalysis`、テスト時は任意。
- `SaveStocks/LoadStocks`、`SaveUserData/LoadUserData/ApplyUserData`（コード単位マージ）、
  `SaveSettings/LoadSettings`。`HasSavedStocks`。例外時は空を返す堅牢実装。

### 3.4 SampleDataService.cs
- `CreateSampleStocks()`：代表 14 銘柄を `Make(...)` で生成し、`BuildHistory` で 10 年時系列を合成、
  `ScoreService.Recalculate` を適用。
- `BuildHistory`：直近売上から成長率で逆算し、利益率・配当・財務・CF・自社株買いを年度展開（乱数で軽い揺らぎ）。

### 3.5 PresetService.cs
- `GetPresets()`：15 種の `ScreeningPreset(name, Func<ScreeningCriteria>)`。
  例「総合評価が最高評価の銘柄」= BuffettScore.Min=75、「株主優待あり高利回り銘柄」= BenefitOnly + BenefitYield.Min。

## 4. ViewModel

### 4.1 共通
- `ViewModelBase : ObservableObject`、`RelayCommand`（`CommandManager.RequerySuggested` 連動）。
- チャート用 DTO（ChartModels.cs）：`ChartBar`/`ChartSeries`/`ChartGroup`/`GaugeItem`。

### 4.2 MainViewModel
- 保有：`ObservableCollection<Stock> AllStocks`、子 VM 3 種、`CurrentViewModel`、`ActiveScreen`、`StatusText`、`DataUpdatedText`。
- コマンド：ShowScreening/ShowAnalysis/ShowComparison、ImportCsv（FilePicker 利用）、LoadSample、Save、Recalculate。
- `LoadInitialData`：保存があれば読み込み、無ければサンプル。ユーザーデータをマージしスコア再計算。
- `ReplaceStocks`：AllStocks 更新後、3 子 VM の `RefreshSourceData` を呼ぶ。
- `OpenAnalysis(stock)`：個別分析へ遷移し対象を選択。`SaveAll`：3 ファイル保存。

### 4.3 ScreeningViewModel
- `Criteria`、`FilteredStocks`、`Presets`、各候補リスト（Sectors/Markets/Scales/BenefitCategories/BenefitMonths）。
- 市場/規模/フラグはトグル用 bool プロパティ（MarketTSE/PR/GR/ST、ScaleSmall/Large/Mid、NoDividendCut/BenefitFlag/CumulativeFlag/DoeFlag）。変更で即 `ApplyFilter`。
- `ApplyFilter`：`Criteria.Matches` で抽出し、バフェット→買いたい度で降順ソート。`ResultText` 更新。
- `ApplyPreset`/`ClearFilter`：Criteria を差し替え、トグル状態を再通知、再フィルタ。
- コマンド：Apply/Clear/ApplyPreset/OpenAnalysis/AddToComparison。

### 4.4 StockAnalysisViewModel
- `StockList`（検索フィルタ）、`SelectedStock`、`Charts`、`Radar`、`BuffettItems`、`YesNoOptions`/`ClassificationOptions`。
- `RebuildDetail`：BuffettItems（15）・Charts（5 群）・Radar（6 軸）を再構築。
- `CheckItem`：回答変更で `OnCheckChanged`→`Recalculate`→レーダー更新。
- `BuildCharts`：系列ごとに最大絶対値で正規化しバー高さを算出（負値は赤）。
- `ApplyChanges`：再計算＋保存。`OpenIr`：IRリンクを既定ブラウザで開く。

### 4.5 ComparisonViewModel
- `AvailableStocks`（検索）、`Selected`（最大6）、`Charts`、`Rows`（旧形式・互換用）。
- `Add/Remove/Clear`→`Rebuild`（テーブルとグラフ更新）。`SelectedCodes`：設定保存用。
- `BuildCharts`：スコア比較／業績・規模／財務／配当・還元の 4 群を銘柄横並びで生成。

## 5. View / 表示ロジック

- App.xaml にコンバーターと VM→View の DataTemplate、`ChartGroup` の簡易バーチャートテンプレートを定義。
- 条件付き書式は `MetricBackgroundConverter`（ConverterParameter にメトリック名）と `MetricRules.Evaluate` で
  Good/Ok/Caution/Bad/Neutral を判定し、半透明色を返す。スコアカードは `ScoreToSolidBrushConverter`。
- フラグは `BoolToFlagConverter`（○ / - / ◎）＋ `FlagToBrushConverter`。グレードは `Grades.Letter`。
- DataGrid は列固定（FrozenColumnCount=2）・行/列仮想化、数値右寄せ、カテゴリ順カラム配置。

## 6. 主要処理シーケンス（CSV 取込）
1. ユーザーが「CSV取込」押下 → MainWindow の FilePicker でファイル選択
2. `CsvImportService.ImportFromFile` で `List<Stock>` 生成
3. `DataStorageService.ApplyUserData` でメモ/チェックをマージ
4. `ScoreService.Recalculate` を全件適用
5. `ReplaceStocks` で AllStocks 更新 → 3 画面へ反映 → `SaveStocks` で永続化
