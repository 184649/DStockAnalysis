## 1. 目的・適用範囲

日本の上場銘柄について、情報・決算結果・財務数値・配当/株主優待・テーマ情報を集約し、ユーザーが
「自分が買いたい銘柄」を見つけるための研究用デスクトップアプリ。証券会社アプリではなく、売買・口座連携・
損益管理・板情報・デイトレード機能は持たない。買い/売りの推奨表現も行わない。

使ううちにバフェット型の投資思考（事業理解・競争優位・収益性・キャッシュ創出力・財務健全性・還元の持続性・
価格妥当性・暴落耐性）が自然に身につく設計とする。

### 適用範囲（本版で実装する機能）
- **全銘柄の自動取得**（JPX 公開「東証上場銘柄一覧」data_j.xls を同梱・起動時自動ロード／月次更新ボタン）
- 全銘柄への擬似指標生成（ユーザーは各指標に絞り込み条件を設定してフィルタ）
- CSV 取り込みによる実データ上書き（銘柄コードで照合）とローカル保存
- 銘柄スクリーニング（条件指定・プリセット・高密度一覧）
- 個別銘柄分析（指標カード・チャート・レーダー・バフェットチェック・株主優待・メモ）
- 銘柄比較（複数銘柄の横並び比較・比較グラフ）
- 各種スコア算出（買い/売り判断は出さない補助指標）

### 適用範囲外（将来拡張）
- J-Quants / EDINET / TDnet 等の外部 API 連携、PDF キーワード検索
- 本格的なチャートライブラリ（LiveCharts2 等）への差し替え

## 2. システム構成

ローカル単独構成。永続化は OS のユーザープロファイル配下の JSON ファイル。

```
                       (任意・月次) JPX 上場銘柄一覧 data_j.xls
                                  │  全銘柄更新(JPX) で再取得
[利用者] ── 操作 ──> [DStockAnalysis.exe (WPF/.NET8)] <── 同梱 Data\data_j.xls(起動時自動ロード)
                           │  CSV 取込（実データ上書き・任意の *.csv）
                           ▼
                  %AppData%\DStockAnalysis\
                    ├ data_j.xls     （更新した最新マスタ・任意）
                    ├ stocks.json    （銘柄データ）
                    ├ userdata.json  （メモ/チェック/興味度）
                    └ settings.json  （比較対象/条件/設定）
```

> JPX 上場銘柄一覧には「コード・銘柄名・市場・業種・規模」のみが含まれ、財務指標は含まれない。
> 本アプリは全銘柄に擬似(サンプル)指標を決定論的に生成し、ユーザーが各指標へ絞り込み条件を設定して
> フィルタリングできるようにする。実データは CSV 取込で上書きする。

### ソフトウェア構成（レイヤ）
| レイヤ | 役割 | 主なクラス |
|---|---|---|
| View | 画面（XAML） | MainWindow, ScreeningView, StockAnalysisView, ComparisonView |
| ViewModel | 画面状態・操作 | MainViewModel, ScreeningViewModel, StockAnalysisViewModel, ComparisonViewModel |
| Model | データ構造 | Stock, TimeSeriesPoint, BuffettCheck, StockMemo, ScreeningCriteria, AppSettings |
| Service | 業務ロジック | JpxMasterService, IndicatorSeedService, CsvImportService, DataStorageService, ScoreService, SampleDataService, PresetService |
| Common/Converters | 共通基盤 | ObservableObject, Grades, 各 IValueConverter |

MVVM とし、View は DataTemplate（App.xaml）で ViewModel に紐づく。View から Service を直接呼ばず、
ViewModel 経由で処理する。ファイル選択ダイアログのみ View 層から ViewModel へ注入（`FilePicker`）。

## 3. 画面設計（概要）

共通：ダーク（Power BI 風）テーマ。上部に画面切替ボタン、右上にデータ更新日・リンク・アイコン領域。
最大化表示前提（1366×768 以上、推奨 1920×1080）。

### 3.1 銘柄スクリーニング画面
- 左：条件指定パネル（業種コンボ、市場/規模/配当還元フラグのトグルボタン、各種数値レンジ最小〜最大）
- 上部：プリセット条件ボタン（15 種）
- 中央：高密度の銘柄一覧テーブル（横スクロール、コード・銘柄名を左固定、カテゴリ順に配置）
- セルは条件付き書式で色分け（良=青/緑、注意=オレンジ、悪=赤、中立=ダークグレー、フラグ=○ / - / ◎）
- 行ダブルクリックまたはボタンで個別分析へ／比較へ追加

### 3.2 個別銘柄分析画面
- 左：銘柄リスト（検索付き）
- ヘッダー：コード/銘柄名/市場/業種/企業概要/決算月/IRリンク/データ更新日/画面切替
- 指標カード：株価・時価総額・PER・PBR・ROE・MIX係数・自己資本比率・配当利回り・配当性向・
  総合評価（S/A/B/C/D）・長期適性・再評価期待・**バフェットスコア（強調）**・買いたい度
- グラフ：業績推移／配当推移／財務推移／キャッシュフロー推移／自己株式の取得（簡易バーチャート）
- スコアレーダー：安全性/成長性/収益性/還元性/効率性/割安性（グレード付き）
- バフェットチェック：15 項目（はい/いいえ/不明）→ スコアに反映
- 株主優待・株主還元エリア、メモ欄（分類・興味度・各種テキスト）

### 3.3 銘柄比較画面
- 左：銘柄選択リスト、上部：比較対象チップ（削除可）
- 中央：選択銘柄を行に並べた比較テーブル（スクリーニングと同列・条件付き書式）
- 下部：比較グラフ（スコア比較／業績・規模／財務／配当・還元）

## 4. データ設計（概要）

### 4.1 主要エンティティ
- **Stock**：1 銘柄の全項目（基本情報・バリュエーション・配当/還元・株主優待・財務・成長性・
  キャッシュフロー・株価変化・スコア・時系列・メモ・バフェットチェック）
- **TimeSeriesPoint**：年度単位の業績・配当・財務・CF・自社株買い
- **BuffettCheck**：15 項目の 3 値（はい/いいえ/不明）
- **StockMemo**：メモ各種＋分類（最重要候補/長期優良株候補/第二のキオクシア候補/再評価候補/
  決算確認待ち/保留/除外）
- **ScreeningCriteria**：絞り込み条件（テキスト・トグル・数値レンジ）
- **AppSettings**：比較対象コード・最終条件・CSV パス

### 4.2 永続化方式
- JSON（System.Text.Json、列挙体は文字列）。`stocks.json` / `userdata.json` / `settings.json`。
- 銘柄本体とユーザー入力（メモ/チェック）を分離し、CSV 再取込時もコード単位で再マージ。

### 4.3 全銘柄マスタ（JPX）
- 取得元：JPX「東証上場銘柄一覧」`data_j.xls`（OLE2/BIFF8、約4,400行）。`Data\data_j.xls` として同梱。
- 抽出：市場・商品区分が プライム/スタンダード/グロース の内国・外国株式のみ（ETF・ETN・REIT・PRO Market・
  出資証券は対象外）。約3,700銘柄。
- 取り込み項目：コード／銘柄名／市場（東証プライム・東証スタンダード・東証グロースへ正規化）／
  33業種区分／規模区分（Core30・Large70→大型、Mid400→中型、Small/その他→小型）／基準日。
- 更新：`UpdateAsync` で最新版を `%AppData%` にダウンロードし再読込（月次更新に対応）。
- 指標：JPX には財務指標が無いため `IndicatorSeedService` が銘柄コードの安定ハッシュ（FNV-1a）から
  決定論的に擬似指標・時系列を生成（再起動しても同一値）。`IsSampleIndicators=true`。

### 4.4 CSV 取り込み（実データ上書き）
- ヘッダー名でマッピング（列順不問・大文字小文字無視）。`Code` 必須。未知/欠損列は 0/空文字。
- ダブルクォート対応（RFC4180 風）。`%`/`,`/`円`/`倍` 等の付与・全角ハイフンを許容。
- `FreeCF`→`FreeCashFlow` 等のエイリアス、`TotalYield`/`MixFactor` の自動補完。
- 取り込み時は銘柄コードで既存銘柄に照合し指標を上書き（`IsSampleIndicators=false`）。未登録コードは追加。

## 5. スコア設計（すべて 0–100、買い/売りは出さない）

| スコア | 主な観点 |
|---|---|
| 6軸（安全性/成長性/収益性/還元性/効率性/割安性） | レーダー表示用 |
| 長期適性スコア | 財務健全性・収益安定性・株主還元・CF・参入障壁・10年後需要 |
| 再評価期待スコア | 売上/営業利益/EPS成長・市場評価の変化・テーマ需要 |
| バフェットスコア | 事業理解/長期需要15・競争優位/参入障壁15・収益性15・キャッシュ創出力15・財務健全性15・株主還元の質10・割安性10・暴落時保有適性5（バフェットチェック反映：はい加点/いいえ減点/不明中立） |
| 買いたい度スコア | 長期適性・再評価・バフェット・割高感・自分の興味・事業理解・メモ分類 |
| 総合評価/総合判定 | 統合し S/A/B/C/D とカテゴリ判定 |

株主優待は還元スコアの加点要素にとどめ、業績不振・配当性向過大・フリーCF 赤字時は加点を抑制。
バフェットスコアでは優待そのものを大きく加点しない。

## 6. 非機能要件
- 性能：数千件の一覧表示でも操作可能（DataGrid の行/列仮想化を有効化）
- 可用性：ローカル単独・無接続で動作。外部障害の影響を受けない
- 保守性：MVVM とレイヤ分離。チャート/データ取得は差し替え可能な構造
- 国際化：日本語 UI（フォント Yu Gothic UI 系）
